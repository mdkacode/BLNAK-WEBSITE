# BLNAK website

Minimalist marketing + blog site for the BLNAK Android launcher. Built with Astro + Tailwind + MDX.
Static, zero-runtime-JS by default, deploys to Vercel.

## Quick start

```bash
cd blnak-website
npm install
npm run dev      # http://localhost:4321
npm run build    # ‚Üí dist/
```

## Adding a blog post

1. Create `src/content/blog/my-post.mdx`
2. Frontmatter:
   ```yaml
   ---
   title: "‚Ä¶"
   description: "‚Ä¶"
   pubDate: 2026-05-01
   tags: ["release-notes"]
   draft: false
   ---
   ```
3. Commit + push. That's the whole workflow ‚Äî posts live in GitHub.

## Things to fill in before launch

- `src/consts.ts` ‚Äî Play Store URL, package name, domain, GitHub URL
- `astro.config.mjs` ‚Äî `site` URL once you buy the domain
- `public/screens/` ‚Äî drop actual app screenshots and swap the 4 placeholders in `src/pages/index.astro`
- `public/og-default.png` ‚Äî 1200√ó630 social preview image
- Replace `favicon.svg` with the final mark

## Compliance checklist (Google Play)

- [x] Privacy policy page, linked from footer
- [x] Data safety disclosures
- [x] Account / data deletion page (required since 2024)
- [x] Terms of service
- [x] Contact email on every policy page
- [x] Last-updated dates on legal pages

## SEO / AI-search

- Per-page `<title>`, `<meta description>`, canonical, OpenGraph, Twitter cards
- JSON-LD: `SoftwareApplication` on home, `BlogPosting` on posts, `FAQPage` on features
- `sitemap-index.xml` (auto-generated by `@astrojs/sitemap`)
- `robots.txt` + `llms.txt` for AI crawlers
- RSS feed at `/rss.xml`

## Deploy to Vercel

1. Push this folder to a GitHub repo.
2. Import into Vercel ‚Äî it auto-detects Astro.
3. Add your custom domain once purchased; set it in `src/consts.ts` + `astro.config.mjs`.
